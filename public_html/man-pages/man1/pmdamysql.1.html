DESCRIPTION

\f3pmdamysql\f1 is a Performance Co-Pilot \s-1PMDA\s0 which extracts live performance data from a running MySQL database.

INSTALLATION

\f3pmdamysql\f1 uses a configuration file from (in this order):

  • /etc/pcpdbi.conf

  • \f(CW$PCP_PMDAS_DIR/mysql/mysql.conf

This file can contain overridden values (Perl code) for the settings listed at the start of pmdamysql.pl, namely:

  • database name (see \s-1DBI\s0\|(3) for details)

  • database username

Once this is setup, you can access the names and values for the mysql performance metrics by doing the following as root:

# cd $PCP_PMDAS_DIR/mysql

# ./Install

To uninstall, do the following as root:

# cd $PCP_PMDAS_DIR/mysql

# ./Remove

pmdamysql is launched by pmcd(1) and should never be executed directly. The Install and Remove scripts notify pmcd(1) when the agent is installed or removed.

Binary Status values in text

Some of the status values are in the form of \s-1YES/NO\s0 or \s-1ON/OFF\s0.

Since these cannot be interpreted by tools like pmie(1), they have been duplicated with a _num extension and the values of 1 (\s-1YES/ON\s0) or 0 (\s-1NO/OFF\s0).

Eg:

  • mysql.slave_status.slave_io_running

  • mysql.slave_status.slave_io_running_num

FILES

\fB/etc/pcpdbi.conf\fR

configuration file for all \s-1PCP\s0 database monitors configuration file for pmdamysql

\fB$PCP_PMDAS_DIR/mysql/Install\fR

installation script for the pmdamysql agent

\fB$PCP_PMDAS_DIR/mysql/Remove\fR

undo installation script for the pmdamysql agent

\fB$PCP_LOG_DIR/pmcd/mysql.log\fR

default log file for error messages from pmdamysql

PCP ENVIRONMENT

Environment variables with the prefix PCP_ are used to parameterize the file and directory names used by PCP. On each installation, the file /etc/pcp.conf contains the local values for these variables. The $PCP_CONF variable may be used to specify an alternative configuration file, as described in pcp.conf(5).
