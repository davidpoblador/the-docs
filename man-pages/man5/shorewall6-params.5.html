SYNOPSIS

/etc/shorewall6/params

DESCRIPTION

Assign any shell variables that you need in this file. The file is always processed by /bin/sh or by the shell specified through SHOREWALL_SHELL in \m[blue]shorewall6.conf\m[]\s-2\u[1]\d\s+2 (5) so the full range of shell capabilities may be used.

It is suggested that variable names begin with an upper case letter to distinguish them from variables used internally within the Shorewall programs

The following variable names must be avoided. Those in bold font must be avoided in all Shorewall versions; those in regular font must be avoided in versions prior to 4.4.8.

Any option from \m[blue]shorewall6.conf\m[]\s-2\u[1]\d\s+2

      (5)

COMMAND

CONFDIR

DEBUG

ECHO_E

ECHO_N

EXPORT

FAST

FILEMODE

HOSTNAME

IPT_OPTIONS

NOROUTES

PREVIEW

PRODUCT

PROFILE

PURGE

RECOVERING

RESTOREPATH

RING_BELL

SHAREDIR

Any name beginning with SHOREWALL_ or

      SW_

STOPPING

TEST

TIMESTAMP

USE_VERBOSITY

VARDIR

VERBOSE

VERBOSE_OFFSET

VERSION

Example params file:

NET_IF=eth0
NET_OPTIONS=dhcp,nosmurfs

Example \m[blue]shorewall6-interfaces\m[]\s-2\u[2]\d\s+2(5) file.

ZONE    INTERFACE       BROADCAST       OPTIONS
net     $NET_IF         -               $NET_OPTIONS

This is the same as if the interfaces file had contained:

ZONE    INTERFACE       BROADCAST       OPTIONS
net     eth0            -               dhcp,nosmurfs
