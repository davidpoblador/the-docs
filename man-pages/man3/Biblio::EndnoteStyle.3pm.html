SYNOPSIS

 use Biblio::EndnoteStyle;
 $style = new Biblio::EndnoteStyle();
 ($text, $errmsg) = $style->format($template, \%fields);

DESCRIPTION

This small module provides a way of formatting bibliographic references using style templates similar to those used by the popular reference management software Endnote (http://www.endnote.com/). The \s-1API\s0 is embarrassingly simple: a formatter object is made using the class's constructor, the \*(C`new()\*(C' method; \*(C`format()\*(C' may then be repeatedly called on this object, using the same or different templates.

(The sole purpose of the object is to cache compiled templates so that multiple \*(C`format()\*(C' invocations are more efficient than they would otherwise be. Apart from that, the \s-1API\s0 might just as well have been a single function.)

METHODS

\$1

$style = new Biblio::EndnoteStyle();

Creates a new formatter object. Takes no arguments. $olddebug = $style->debug(1);

Turns debugging on or off and returns the old debugging status. If an argument is provided, then debugging is turned either on or off according to whether then argument is true or false. In any case, the old value of the debugging status is returned, so that a call with no argument is a side-effect-free inquiry.

When debugging is turned on, compiled templates are dumped to standard error. It is not pretty. ($text, $errmsg) = $style->format($template, \%fields);

Formats a reference, consisting of a hash of fields, according to an Endnote-like template. The template is a string essentially the same as those used in Endnote, as documented in the Endnote X User Guide at http://www.endnote.com/support/helpdocs/EndNoteXWinManual.pdf pages 390ff. In particular, pages 415-210 have details of the recipe format. Because the templates used in this module are plain text, a few special characters are used:

A\*^X

Link adjacent words. This is the \*(L"non-breaking space\*(R" described on page 418 of the EndNote X

|

Forced Seperation of elements that would otherwise be dependent.

^

Separator for singular/plural aternatives.

`

Used to prevent literal text from being interpreted as a fieldname.

The hash of fields is passed by reference: keys are fieldnames, and the corresponding values are the data. \s-1PLEASE\s0 \s-1NOTE\s0 \s-1AN\s0 \s-1IMPORTANT\s0 \s-1DIFFERENCE\s0. Keys that do not appear in the hash at all are not considered to be fields, so that if they appear in the template, they will be interpreted as literal text; keys that appear in the hash but whose values are undefined or empty are considered to be fields with no value, and will be formatted as empty with dependent text omitted. So for example:

$style->format(";Author: ", { Author => "Taylor" }) eq ":Taylor: " $style->format(";Author: ", { Author => "" }) eq ";" $style->format(";Author: ", { xAuthor => "" }) eq ";Author: "

\*(C`format()\*(C' returns two values: the formatted reference and an error-message. The error message is defined if and only if the formatted reference is not.
