  # ...

  if($resp->is_success()) {
      for my $prop ($resp->properties) {
          print join("/", $_->artists(), " ",
                $_->title(), " ",
                $_->year(), " ";
                $_->running_time(), " ";
   }

DESCRIPTION

\*(C`Net::Amazon::Property::DigitalMusicTrack\*(C' is derived from \*(C`Net::Amazon::Property\*(C' and on top of the all-purpose methods the base class provides, it offers specialized accessors for popular digital music track parameters.

\s-1METHODS\s0

artists()

Returns a list of the artists. There's also a \*(C`artist()\*(C' method which just returns the first artist.

binding()

Returns the binding of the digital music track, such as \s-1MP3\s0 Download.

genre()

Returns the genre as a string.

label()

Returns the music label as a string.

manufacturer()

Returns the manufacturer as a string.

publisher()

Returns the publisher as a string.

release_date()

Returns the release date as a string formatted YYYY-MM-DD.

studio()

Returns the studio as a string.

title()

Returns the title as a string.

track_sequence()

Returns the track sequence number of this song from its released \s-1CD\s0.

running_time()

Running time of this track in seconds. Initializes an object by passing a hash of hashes structure containing the \s-1XML\s0 data returned from the service. Usually, this is just used by \*(C`Net::Amazon\*(C' internally to initialize objects for on backcoming data.

Check out Net::Amazon::Property for all-purpose accessors, like \*(C`year\*(C', \*(C`OurPrice\*(C', \*(C`ListPrice\*(C', etc.
