INHERITANCE

  MDOM::Element is the root of the PDOM tree

DESCRIPTION

The abstract \*(C`MDOM::Element\*(C' serves as a base class for all source-related objects, from a single whitespace token to an entire document. It provides a basic set of methods to provide a common interface and basic implementations.

METHODS

significant

Because we treat whitespace and other non-code items as Tokens (in order to be able to \*(L"round trip\*(R" the MDOM::Document back to a file) the \*(C`significant\*(C' method allows us to distinguish between tokens that form a part of the code, and tokens that aren't significant, such as whitespace, \s-1POD\s0, or the portion of a file after (and including) the \*(C`_\|_END_\|_\*(C' token.

Returns true if the Element is significant, or false it not.

class

The \*(C`class\*(C' method is provided as a convenience, and really does nothing more than returning \*(C`ref($self)\*(C'. However, some people have found that they appreciate the laziness of \*(C`$Foo->class eq 'whatever'\*(C', so I have caved to popular demand and included it.

Returns the class of the Element as a string

tokens

The \*(C`tokens\*(C' method returns a list of MDOM::Token objects for the Element, essentially getting back that part of the document as if it had not been lexed.

This also means there are no Statements and no Structures in the list, just the Token classes.

content

For any \*(C`MDOM::Element\*(C', the \*(C`content\*(C' method will reconstitute the base code for it as a single string. This method is also the method used for overloading stringification. When an Element is used in a double-quoted string for example, this is the method that is called.

\s-1WARNING:\s0

You should be aware that because of the way that here-docs are handled, any here-doc content is not included in \*(C`content\*(C', and as such you should not eval or execute the result if it contains any MDOM::Token::HereDoc.

The MDOM::Document method \*(C`serialize\*(C' should be used to stringify a \s-1PDOM\s0 document into something that can be executed as expected.

Returns the basic code as a string (excluding here-doc content).

parent

Elements themselves are not intended to contain other Elements, that is left to the MDOM::Node abstract class, a subclass of \*(C`MDOM::Element\*(C'. However, all Elements can be contained within a parent Node.

If an Element is within a parent Node, the \*(C`parent\*(C' method returns the Node.

statement

For a \*(C`MDOM::Element\*(C' that is contained (at some depth) within a MDOM::Statment, the \*(C`statement\*(C' method will return the first parent Statement object lexically 'above' the Element.

Returns a MDOM::Statement object, which may be the same Element if the Element is itself a MDOM::Statement object.

Returns false if the Element is not within a Statement and is not itself a Statement.

top

For a \*(C`MDOM::Element\*(C' that is contained within a \s-1PDOM\s0 tree, the \*(C`top\*(C' method will return the top-level Node in the tree. Most of the time this should be a MDOM::Document object, however this will not always be so. For example, if a subroutine has been removed from its Document, to be moved to another Document.

Returns the top-most \s-1PDOM\s0 object, which may be the same Element, if it is not within any parent \s-1PDOM\s0 object.

For an Element that is contained within a MDOM::Document object, the \*(C`document\*(C' method will return the top-level Document for the Element.

Returns the MDOM::Document for this Element, or false if the Element is not contained within a Document.

next_sibling

All MDOM::Node objects (specifically, our parent Node) contain a number of \*(C`MDOM::Element\*(C' objects. The \*(C`next_sibling\*(C' method returns the \*(C`MDOM::Element\*(C' immediately after the current one, or false if there is no next sibling.

snext_sibling

As per the other 's' methods, the \*(C`snext_sibling\*(C' method returns the next significant sibling of the \*(C`MDOM::Element\*(C' object.

Returns a \*(C`MDOM::Element\*(C' object, or false if there is no 'next' significant sibling.

previous_sibling

All MDOM::Node objects (specifically, our parent Node) contain a number of \*(C`MDOM::Element\*(C' objects. The \*(C`previous_sibling\*(C' method returns the Element immediately before the current one, or false if there is no 'previous' \*(C`MDOM::Element\*(C' object.

sprevious_sibling

As per the other 's' methods, the \*(C`sprevious_sibling\*(C' method returns the previous significant sibling of the \*(C`MDOM::Element\*(C' object.

Returns a \*(C`MDOM::Element\*(C' object, or false if there is no 'previous' significant sibling.

first_token

As a support method for higher-order algorithms that deal specifically with tokens and actual Perl content, the \*(C`first_token\*(C' method finds the first MDOM::Token object within or equal to this one.

That is, if called on a MDOM::Node subclass, it will descend until it finds a MDOM::Token. If called on a MDOM::Token object, it will return the same object.

Returns a MDOM::Token object, or dies on error (which should be extremely rare and only occur if an illegal empty MDOM::Statement exists below the current Element somewhere.

last_token

As a support method for higher-order algorithms that deal specifically with tokens and actual Perl content, the \*(C`last_token\*(C' method finds the last MDOM::Token object within or equal to this one.

That is, if called on a MDOM::Node subclass, it will descend until it finds a MDOM::Token. If called on a MDOM::Token object, it will return the itself.

Returns a MDOM::Token object, or dies on error (which should be extremely rare and only occur if an illegal empty MDOM::Statement exists below the current Element somewhere.

next_token

As a support method for higher-order algorithms that deal specifically with tokens and actual Perl content, the \*(C`next_token\*(C' method finds the MDOM::Token object that is immediately after the current Element, even if it is not within the same parent MDOM::Node as the one for which the method is being called.

Note that this is not defined as a MDOM::Token-specific method, because it can be useful to find the next token that is after, say, a MDOM::Statement, although obviously it would be useless to want the next token after a MDOM::Document.

Returns a MDOM::Token object, or false if there are no more tokens after the Element.

previous_token

As a support method for higher-order algorithms that deal specifically with tokens and actual Perl content, the \*(C`previous_token\*(C' method finds the MDOM::Token object that is immediately before the current Element, even if it is not within the same parent MDOM::Node as this one.

Note that this is not defined as a MDOM::Token-only method, because it can be useful to find the token is before, say, a MDOM::Statement, although obviously it would be useless to want the next token before a MDOM::Document.

Returns a MDOM::Token object, or false if there are no more tokens before the \*(C`Element\*(C'.

clone

As per the Clone module, the \*(C`clone\*(C' method makes a perfect copy of an Element object. In the generic case, the implementation is done using the Clone module's mechanism itself. In higher-order cases, such as for Nodes, there is more work involved to keep the parent-child links intact. The \*(C`insert_before\*(C' method allows you to insert lexical perl content, in the form of \*(C`MDOM::Element\*(C' objects, before the calling \*(C`Element\*(C'. You need to be very careful when modifying perl code, as it's easy to break things.

In its initial incarnation, this method allows you to insert a single Element, and will perform some basic checking to prevent you inserting something that would be structurally wrong (in \s-1PDOM\s0 terms).

In future, this method may be enhanced to allow the insertion of multiple Elements, inline-parsed code strings or MDOM::Document::Fragment objects.

Returns true if the Element was inserted, false if it can not be inserted, or \*(C`undef\*(C' if you do not provide a MDOM::Element object as a parameter. The \*(C`insert_after\*(C' method allows you to insert lexical perl content, in the form of \*(C`MDOM::Element\*(C' objects, after the calling \*(C`Element\*(C'. You need to be very careful when modifying perl code, as it's easy to break things.

Returns true if the Element was inserted, false if it can not be inserted, or \*(C`undef\*(C' if you do not provide a MDOM::Element object as a parameter.

remove

For a given \*(C`MDOM::Element\*(C', the \*(C`remove\*(C' method will remove it from its parent intact, along with all of its children.

Returns the \*(C`Element\*(C' itself as a convenience, or \*(C`undef\*(C' if an error occurs while trying to remove the \*(C`Element\*(C'.

delete

For a given \*(C`MDOM::Element\*(C', the \*(C`remove\*(C' method will remove it from its parent, immediately deleting the \*(C`Element\*(C' and all of its children (if it has any).

Returns true if the \*(C`Element\*(C' was successfully deleted, or \*(C`undef\*(C' if an error occurs while trying to remove the \*(C`Element\*(C'. Although some higher level class support more exotic forms of replace, at the basic level the \*(C`replace\*(C' method takes a single \*(C`Element\*(C' as an argument and replaces the current \*(C`Element\*(C' with it.

To prevent accidental damage to code, in this initial implementation the replacement element must be of the same class (or a subclass) as the one being replaced.

location

If the Element exists within a MDOM::Document that has indexed the Element locations using \*(C`MDOM::Document::index_locations\*(C', the \*(C`location\*(C' method will return the location of the first character of the Element within the Document.

Returns the location as a reference to a three-element array in the form \*(C`[ $line, $rowchar, $col ]\*(C'. The values are in a human format, with the first character of the file located at \*(C`[ 1, 1, 1 ]\*(C'.

The second and third numbers are similar, except that the second is the literal horizontal character, and the third is the visual column, taking into account tabbing.

Returns \*(C`undef\*(C' on error, or if the MDOM::Document object has not been indexed.

TO DO

It would be nice if \*(C`location\*(C' could be used in an ad-hoc manner. That is, if called on an Element within a Document that has not been indexed, it will do a one-off calculation to find the location. It might be very painful if someone started using it a lot, without remembering to index the document, but it would be handy for things that are only likely to use it once, such as error handlers.
