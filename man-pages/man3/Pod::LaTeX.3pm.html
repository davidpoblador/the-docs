SYNOPSIS

  use Pod::LaTeX;
  my $parser = Pod::LaTeX->new ( );

  $parser->parse_from_filehandle;

  $parser->parse_from_file ('file.pod', 'file.tex');

DESCRIPTION

\*(C`Pod::LaTeX\*(C' is a module to convert documentation in the Pod format into Latex. The pod2latex command uses this module for translation.

\*(C`Pod::LaTeX\*(C' is a derived class from Pod::Select.

OBJECT METHODS

The following methods are provided in this module. Methods inherited from \*(C`Pod::Select\*(C' are not described in the public interface.

Data Accessors

The following methods are provided for accessing instance data. These methods should be used for accessing configuration parameters rather than assuming the object is a hash.

Default values can be supplied by using these names as keys to a hash of arguments when using the \*(C`new()\*(C' constructor.

AddPreamble

Logical to control whether a \*(C`latex\*(C' preamble is to be written. If true, a valid \*(C`latex\*(C' preamble is written before the pod data is written. This is similar to: \documentclass{article} \usepackage[T1]{fontenc} \usepackage{textcomp} \begin{document} but will be more complicated if table of contents and indexing are required. Can be used to set or retrieve the current value. $add = $parser->AddPreamble(); $parser->AddPreamble(1); If used in conjunction with \*(C`AddPostamble\*(C' a full latex document will be written that could be immediately processed by \*(C`latex\*(C'. For some pod escapes it may be necessary to include the amsmath package. This is not yet added to the preamble automatically.

AddPostamble

Logical to control whether a standard \*(C`latex\*(C' ending is written to the output file after the document has been processed. In its simplest form this is simply: \end{document} but can be more complicated if a index is required. Can be used to set or retrieve the current value. $add = $parser->AddPostamble(); $parser->AddPostamble(1); If used in conjunction with \*(C`AddPreaamble\*(C' a full latex document will be written that could be immediately processed by \*(C`latex\*(C'.

Head1Level

The \*(C`latex\*(C' sectioning level that should be used to correspond to a pod \*(C`=head1\*(C' directive. This can be used, for example, to turn a \*(C`=head1\*(C' into a \*(C`latex\*(C' \*(C`subsection\*(C'. This should hold a number corresponding to the required position in an array containing the following elements: [0] chapter [1] section [2] subsection [3] subsubsection [4] paragraph [5] subparagraph Can be used to set or retrieve the current value: $parser->Head1Level(2); $sect = $parser->Head1Level; Setting this number too high can result in sections that may not be reproducible in the expected way. For example, setting this to 4 would imply that \*(C`=head3\*(C' do not have a corresponding \*(C`latex\*(C' section (\*(C`=head1\*(C' would correspond to a \*(C`paragraph\*(C'). A check is made to ensure that the supplied value is an integer in the range 0 to 5. Default is for a value of 1 (i.e. a \*(C`section\*(C').

Label

This is the label that is prefixed to all \*(C`latex\*(C' label and index entries to make them unique. In general, pods have similarly titled sections (\s-1NAME\s0, \s-1DESCRIPTION\s0 etc) and a \*(C`latex\*(C' label will be multiply defined if more than one pod document is to be included in a single \*(C`latex\*(C' file. To overcome this, this label is prefixed to a label whenever a label is required (joined with an underscore) or to an index entry (joined by an exclamation mark which is the normal index separator). For example, \*(C`\label{text}\*(C' becomes \*(C`\label{Label_text}\*(C'. Can be used to set or retrieve the current value: $label = $parser->Label; $parser->Label($label); This label is only used if \*(C`UniqueLabels\*(C' is true. Its value is set automatically from the \*(C`NAME\*(C' field if \*(C`ReplaceNAMEwithSection\*(C' is true. If this is not the case it must be set manually before starting the parse. Default value is \*(C`undef\*(C'.

LevelNoNum

Control the point at which \*(C`latex\*(C' section numbering is turned off. For example, this can be used to make sure that \*(C`latex\*(C' sections are numbered but subsections are not. Can be used to set or retrieve the current value: $lev = $parser->LevelNoNum; $parser->LevelNoNum(2); The argument must be an integer between 0 and 5 and is the same as the number described in \*(C`Head1Level\*(C' method description. The number has nothing to do with the pod heading number, only the \*(C`latex\*(C' sectioning. Default is 2. (i.e. \*(C`latex\*(C' subsections are written as \*(C`subsection*\*(C' but sections are numbered).

MakeIndex

Controls whether \*(C`latex\*(C' commands for creating an index are to be inserted into the preamble and postamble $makeindex = $parser->MakeIndex; $parser->MakeIndex(0); Irrelevant if both \*(C`AddPreamble\*(C' and \*(C`AddPostamble\*(C' are false (or equivalently, \*(C`UserPreamble\*(C' and \*(C`UserPostamble\*(C' are set). Default is for an index to be created.

ReplaceNAMEwithSection

This controls whether the \*(C`NAME\*(C' section in the pod is to be translated literally or converted to a slightly modified output where the section name is the pod name rather than \*(L"\s-1NAME\s0\*(R". If true, the pod segment =head1 NAME

pod::name - purpose

=head1 SYNOPSIS is converted to the \*(C`latex\*(C' \section{pod::name\label{pod_name}\index{pod::name}}

Purpose

\subsection*{SYNOPSIS\label{pod_name_SYNOPSIS}% \index{pod::name!SYNOPSIS}} (dependent on the value of \*(C`Head1Level\*(C' and \*(C`LevelNoNum\*(C'). Note that subsequent \*(C`head1\*(C' directives translate to subsections rather than sections and that the labels and index now include the pod name (dependent on the value of \*(C`UniqueLabels\*(C'). The \*(C`Label\*(C' is set from the pod name regardless of any current value of \*(C`Label\*(C'. $mod = $parser->ReplaceNAMEwithSection; $parser->ReplaceNAMEwithSection(0); Default is to translate the pod literally.

StartWithNewPage

If true, each pod translation will begin with a \*(C`latex\*(C' \*(C`\clearpage\*(C'. $parser->StartWithNewPage(1); $newpage = $parser->StartWithNewPage; Default is false.

TableOfContents

If true, a table of contents will be created. Irrelevant if \*(C`AddPreamble\*(C' is false or \*(C`UserPreamble\*(C' is set. $toc = $parser->TableOfContents; $parser->TableOfContents(1); Default is false.

UniqueLabels

If true, the translator will attempt to make sure that each \*(C`latex\*(C' label or index entry will be uniquely identified by prefixing the contents of \*(C`Label\*(C'. This allows multiple documents to be combined without clashing common labels such as \*(C`DESCRIPTION\*(C' and \*(C`SYNOPSIS\*(C' $parser->UniqueLabels(1); $unq = $parser->UniqueLabels; Default is true.

UserPreamble

User supplied \*(C`latex\*(C' preamble. Added before the pod translation data. If set, the contents will be prepended to the output file before the translated data regardless of the value of \*(C`AddPreamble\*(C'. \*(C`MakeIndex\*(C' and \*(C`TableOfContents\*(C' will also be ignored.

UserPostamble

User supplied \*(C`latex\*(C' postamble. Added after the pod translation data. If set, the contents will be prepended to the output file after the translated data regardless of the value of \*(C`AddPostamble\*(C'. \*(C`MakeIndex\*(C' will also be ignored.

NOTES

Compatible with \*(C`latex2e\*(C' only. Can not be used with \*(C`latex\*(C' v2.09 or earlier.

A subclass of \*(C`Pod::Select\*(C' so that specific pod sections can be converted to \*(C`latex\*(C' by using the \*(C`select\*(C' method.

Some \s-1HTML\s0 escapes are missing and many have not been tested.
