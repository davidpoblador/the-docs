SYNOPSIS

 use XML::Grove::Factory;

 ### An object that creates Grove objects directly
 my $gf = XML::Grove::Factory->grove_factory;

 $grove = $gf->document( CONTENTS );
 $element = $gf->element( $name, { ATTRIBUTES }, CONTENTS );
 $pi = $gf->pi( $target, $data );
 $comment = $gf->comment( $data );

 ### An object that creates elements by method name
 my $ef = XML::Grove::Factory->element_factory();

 $element = $ef->NAME( { ATTRIBUTES }, CONTENTS);

 ### Similar to `element_factory', but creates functions in the
 ### current package
 XML::Grove::Factory->element_functions( PREFIX, ELEMENTS );

 $element = NAME( { ATTRIBUTES }, CONTENTS );

DESCRIPTION

\*(C`XML::Grove::Factory\*(C' provides objects or defines functions that let you simply and quickly create the most commonly used XML::Grove objects. \*(C`XML::Grove::Factory\*(C' supports three types of object creation. The first type is to create raw XML::Grove objects. The second type creates \s-1XML\s0 elements by element name. The third type is like the second, but defines local functions for you to call instead of using an object, which might save typing in some cases.

The three types of factories can be mixed. For example, you can use local functions for all element names that don't conflict with your own sub names or contain special characters, and then use a `\*(C`grove_factory()\*(C'' object for those elements that do conflict.

In the examples that follow, each example is creating an \s-1XML\s0 instance similar to the following, assuming it's pretty printed:

<?xml version="1.0"?> <HTML> <HEAD> <TITLE>Some Title</TITLE> </HEAD> <BODY bgcolor="#FFFFFF"> <P>A paragraph.</P> </BODY> </HTML>

GROVE FACTORY

Creates a new grove factory object that creates raw XML::Grove objects. Creates an XML::Grove::Document object. \s-1CONTENTS\s0 may contain processing instructions, strings containing only whitespace characters, and a single element object (but note that there is no checking). Strings are converted to XML::Grove::Characters objects. Creates an XML::Grove::Element object with the name `$name'. If the argument following `$name' is an anonymous hash, \s-1ATTRIBUTES\s0, then they will be copied to the elements attributes. \s-1CONTENTS\s0 will be stored in the element's content (note that there is no validity checking). Strings in \s-1CONTENTS\s0 are converted to XML::Grove::Characters objects. Create an XML::Grove::PI object with \s-1TARGET\s0 and \s-1DATA\s0. Create an XML::Grove::Comment object with \s-1DATA\s0.

\s-1GROVE\s0 \s-1FACTORY\s0 \s-1EXAMPLE\s0

use XML::Grove::Factory;

$gf = XML::Grove::Factory->grove_factory;

$element = $gf->element('HTML', $gf->element('HEAD', $gf->element('TITLE', 'Some Title')), $gf->element('BODY', { bgcolor => '#FFFFFF' }, $gf->element('P', 'A paragraph.')));

ELEMENT FACTORY

Creates a new element factory object for creating elements. `\*(C`element_factory()\*(C'' objects work by creating an element for any name used to call the object. Creates an XML::Grove::Element object with the given \s-1NAME\s0, \s-1ATTRIBUTES\s0, and \s-1CONTENTS\s0. The hash containing \s-1ATTRIBUTES\s0 is optional if this element doesn't need attributes. Strings in \s-1CONTENTS\s0 are converted to XML::Grove::Characters objects.

\s-1ELEMENT\s0 \s-1FACTORY\s0 \s-1EXAMPLE\s0

use XML::Grove::Factory;

$ef = XML::Grove::Factory->element_factory();

$element = $ef->HTML( $ef->HEAD( $ef->TITLE('Some Title')), $ef->BODY({ bgcolor => '#FFFFFF' }, $ef->P('A paragraph.')));

ELEMENT FUNCTIONS

XML::Grove::Factory->element_functions (\s-1PREFIX\s0, \s-1ELEMENTS\s0)

Creates functions in the current package for creating elements with the names provided in the list \s-1ELEMENTS\s0. \s-1PREFIX\s0 will be prepended to every function name, or \s-1PREFIX\s0 can be an empty string ('') if you're confident that there won't be any conflicts with functions in your package.

\s-1NAME\s0( \s-1CONTENTS\s0 )
\s-1NAME\s0( { \s-1ATTRIBUTES\s0 }, \s-1CONTENTS\s0 )
\s-1PREFIX\s0\s-1NAME\s0( \s-1CONTENTS\s0 )
\s-1PREFIX\s0\s-1NAME\s0( { \s-1ATTRIBUTES\s0 }, \s-1CONTENTS\s0 )

Functions created for `\*(C`NAME\*(C'' or `\*(C`PREFIXNAME\*(C'' can be called to create XML::Grove::Element objects with the given \s-1NAME\s0, \s-1ATTRIBUTES\s0, and \s-1CONTENT\s0. The hash containing \s-1ATTRIBUTES\s0 is optional if this element doesn't need attributes. Strings in \s-1CONTENT\s0 are converted to XML::Grove::Characters objects.

\s-1ELEMENT\s0 \s-1FACTORY\s0 \s-1EXAMPLE\s0

use XML::Grove::Factory;

XML::Grove::Factory->element_functions('', qw{ HTML HEAD TITLE BODY P });

$element = HTML( HEAD( TITLE('Some Title')), BODY({ bgcolor => '#FFFFFF' }, P('A paragraph.')));
