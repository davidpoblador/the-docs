SYNOPSIS

        #!perl -w
        use MP3::Info;
        my $file = 'Pearls_Before_Swine.mp3';
        set_mp3tag($file, 'Pearls Before Swine', q"77's",
                'Sticks and Stones', '1990',
                q"(c) 1990 77's LTD.", 'rock & roll');

        my $tag = get_mp3tag($file) or die "No TAG info";
        $tag->{GENRE} = 'rock';
        set_mp3tag($file, $tag);

        my $info = get_mp3info($file);
        printf "$file length is %d:%d\n", $info->{MM}, $info->{SS};

my $mp3 = new MP3::Info $file; $mp3->title('Perls Before Swine'); printf "$file length is %s, title is %s\n", $mp3->time, $mp3->title;

DESCRIPTION

\s-1OOP\s0 interface to the rest of the module. The same keys available via get_mp3info and get_mp3tag are available via the returned object (using upper case or lower case; but note that all-caps \*(L"\s-1VERSION\s0\*(R" will return the module version, not the \s-1MP3\s0 version). Passing a value to one of the methods will set the value for that tag in the \s-1MP3\s0 file, if applicable.

use_mp3_utf8([\s-1STATUS\s0])

Tells MP3::Info to (or not) return \s-1TAG\s0 info in \s-1UTF-8\s0. \s-1TRUE\s0 is 1, \s-1FALSE\s0 is 0. Default is \s-1TRUE\s0, if available. Will only be able to turn it on if Encode is available. ID3v2 tags will be converted to \s-1UTF-8\s0 according to the encoding specified in each tag; ID3v1 tags will be assumed Latin-1 and converted to \s-1UTF-8\s0. Function returns status (\s-1TRUE/FALSE\s0). If no argument is supplied, or an unaccepted argument is supplied, function merely returns status. This function is not exported by default, but may be exported with the \*(C`:utf8\*(C' or \*(C`:all\*(C' export tag.

use_winamp_genres()

Puts WinAmp genres into @mp3_genres and %mp3_genres (adds 68 additional genres to the default list of 80). This is a separate function because these are non-standard genres, but they are included because they are widely used. You can import the data structures with one of: use MP3::Info qw(:genres); use MP3::Info qw(:DEFAULT :genres); use MP3::Info qw(:all);

remove_mp3tag (\s-1FILE\s0 [, \s-1VERSION\s0, \s-1BUFFER\s0])

Can remove ID3v1 or ID3v2 tags. \s-1VERSION\s0 should be 1 for ID3v1 (the default), 2 for ID3v2, and \*(C`ALL\*(C' for both. For ID3v1, removes last 128 bytes from file if those last 128 bytes begin with the text '\s-1TAG\s0'. File will be 128 bytes shorter. For ID3v2, removes ID3v2 tag. Because an ID3v2 tag is at the beginning of the file, we rewrite the file after removing the tag data. The buffer for rewriting the file is 4MB. \s-1BUFFER\s0 (in bytes) ca change the buffer size. Returns the number of bytes removed, or -1 if no tag removed, or undef if there is an error.

set_mp3tag (\s-1FILE\s0, \s-1TITLE\s0, \s-1ARTIST\s0, \s-1ALBUM\s0, \s-1YEAR\s0, \s-1COMMENT\s0, \s-1GENRE\s0 [, \s-1TRACKNUM\s0])

Adds/changes tag information in an \s-1MP3\s0 audio file. Will clobber any existing information in file. Fields are \s-1TITLE\s0, \s-1ARTIST\s0, \s-1ALBUM\s0, \s-1YEAR\s0, \s-1COMMENT\s0, \s-1GENRE\s0. All fields have a 30-byte limit, except for \s-1YEAR\s0, which has a four-byte limit, and \s-1GENRE\s0, which is one byte in the file. The \s-1GENRE\s0 passed in the function is a case-insensitive text string representing a genre found in @mp3_genres. Will accept either a list of values, or a hashref of the type returned by \*(C`get_mp3tag\*(C'. If \s-1TRACKNUM\s0 is present (for ID3v1.1), then the \s-1COMMENT\s0 field can only be 28 bytes. ID3v2 support may come eventually. Note that if you set a tag on a file with ID3v2, the set tag will be for ID3v1[.1] only, and if you call \*(C`get_mp3tag\*(C' on the file, it will show you the (unchanged) ID3v2 tags, unless you specify ID3v1.

get_mp3tag (\s-1FILE\s0 [, \s-1VERSION\s0, \s-1RAW_V2\s0, \s-1APE2\s0])

Returns hash reference containing tag information in \s-1MP3\s0 file. The keys returned are the same as those supplied for \*(C`set_mp3tag\*(C', except in the case of \s-1RAW_V2\s0 being set. If \s-1VERSION\s0 is 1, the information is taken from the ID3v1 tag (if present). If \s-1VERSION\s0 is 2, the information is taken from the ID3v2 tag (if present). If \s-1VERSION\s0 is not supplied, or is false, the ID3v1 tag is read if present, and then, if present, the ID3v2 tag information will override any existing ID3v1 tag info. If \s-1RAW_V2\s0 is 1, the raw ID3v2 tag data is returned, without any manipulation of text encoding. The key name is the same as the frame \s-1ID\s0 (\s-1ID\s0 to name mappings are in the global %v2_tag_names). If \s-1RAW_V2\s0 is 2, the ID3v2 tag data is returned, manipulating for Unicode if necessary, etc. It also takes multiple values for a given key (such as comments) and puts them in an arrayref. If \s-1APE\s0 is 1, an \s-1APE\s0 tag will be located before all other tags. If the ID3v2 version is older than ID3v2.2.0 or newer than ID3v2.4.0, it will not be read. Strings returned will be in Latin-1, unless \s-1UTF-8\s0 is specified (use_mp3_utf8), (unless \s-1RAW_V2\s0 is 1). Also returns a \s-1TAGVERSION\s0 key, containing the \s-1ID3\s0 version used for the returned data (if \s-1TAGVERSION\s0 argument is 0, may contain two versions).

get_mp3info (\s-1FILE\s0)

Returns hash reference containing file information for \s-1MP3\s0 file. This data cannot be changed. Returned data: VERSION MPEG audio version (1, 2, 2.5) LAYER MPEG layer description (1, 2, 3) STEREO boolean for audio is in stereo

VBR boolean for variable bitrate BITRATE bitrate in kbps (average for VBR files) FREQUENCY frequency in kHz SIZE bytes in audio stream OFFSET bytes offset that stream begins

SECS total seconds MM minutes SS leftover seconds MS leftover milliseconds TIME time in MM:SS

COPYRIGHT boolean for audio is copyrighted PADDING boolean for MP3 frames are padded MODE channel mode (0 = stereo, 1 = joint stereo, 2 = dual channel, 3 = single channel) FRAMES approximate number of frames FRAME_LENGTH approximate length of a frame VBR_SCALE VBR scale from VBR header On error, returns nothing and sets $@.

If you find a bug, please send me a patch (see the project page in \*(L"\s-1SEE\s0 \s-1ALSO\s0\*(R"). If you cannot figure out why it does not work for you, please put the \s-1MP3\s0 file in a place where I can get it (preferably via \s-1FTP\s0, or \s-1HTTP\s0, or .Mac iDisk) and send me mail regarding where I can get the file, with a detailed description of the problem.

If I download the file, after debugging the problem I will not keep the \s-1MP3\s0 file if it is not legal for me to have it. Just let me know if it is legal for me to keep it or not.
