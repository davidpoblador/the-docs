SYNOPSIS

    use Regexp::Common qw /comment/;

    while (<>) {
        /$RE{comment}{C}/       and  print "Contains a C comment\n";
        /$RE{comment}{C++}/     and  print "Contains a C++ comment\n";
        /$RE{comment}{PHP}/     and  print "Contains a PHP comment\n";
        /$RE{comment}{Java}/    and  print "Contains a Java comment\n";
        /$RE{comment}{Perl}/    and  print "Contains a Perl comment\n";
        /$RE{comment}{awk}/     and  print "Contains an awk comment\n";
        /$RE{comment}{HTML}/    and  print "Contains an HTML comment\n";
    }

    use Regexp::Common qw /comment RE_comment_HTML/;

    while (<>) {
        $_ =~ RE_comment_HTML() and  print "Contains an HTML comment\n";
    }

DESCRIPTION

Please consult the manual of Regexp::Common for a general description of the works of this interface.

Do not use this module directly, but load it via Regexp::Common.

This modules gives you regular expressions for comments in various languages.

\s-1THE\s0 \s-1LANGUAGES\s0

Below, the comments of each of the languages are described. The patterns are available as $RE{comment}{LANG}, foreach language \s-1LANG\s0. Some languages have variants; it's described at the individual languages how to get the patterns for the variants. Unless mentioned otherwise, \*(C`{-keep}\*(C' sets $1, $2, $3 and $4 to the entire comment, the opening marker, the content of the comment, and the closing marker (for many languages, the latter is a newline) respectively.

Note that this is not the same as something that starts with \*(C`<!--\*(C' and ends with \*(C`-->\*(C', because the following will be matched completely: <!-- First Comment -- --> Second Comment <!-- -- Third Comment --> Do not be fooled by what your favourite browser thinks is an \s-1HTML\s0 comment. If \*(C`{-keep}\*(C' is used, the following are returned:

captures the entire comment declaration. captures the \s-1MDO\s0 (markup declaration open), \*(C`<!\*(C'. captures the content between the \s-1MDO\s0 and the \s-1MDC\s0. captures the (last) comment, without the surrounding dashes. captures the \s-1MDC\s0 (markup declaration close), \*(C`>\*(C'.

There are many implementations of Pascal. This modules provides pattern for comments of several implementations.

This is the pattern that recognizes comments according to the Pascal \s-1ISO\s0 standard. The Delphi Pascal, Free Pascal and the Gnu Pascal Compiler implementations of Pascal all have comments that either start with \*(C`//\*(C' and last till the end of the line, are delimited with \*(C`{\*(C' and \*(C`}\*(C' or are delimited with \*(C`(*\*(C' and \*(C`*)\*(C'. These patterns only set $1 when \*(C`{-keep}\*(C' is used, which will then include the entire comment. See <http://info.borland.com/techpubs/delphi5/oplg/>, http://www.freepascal.org/docs-html/ref/ref.html <http://www.freepascal.org/docs-html/ref/ref.html> and http://www.gnu-pascal.de/gpc/ <http://www.gnu-pascal.de/gpc/>.
