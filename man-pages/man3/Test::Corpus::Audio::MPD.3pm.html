SYNOPSIS

    use Test::Corpus::Audio::MPD; # die if error
    [...]
    stop_test_mpd();

DESCRIPTION

This module will try to launch a new mpd server for testing purposes. This mpd server will then be used during POE::Component::Client::MPD or Audio::MPD tests.

In order to achieve this, the module will create a fake mpd.conf file with the correct pathes (ie, where you untarred the module tarball). It will then check if some mpd server is already running, and stop it if the \*(C`MPD_TEST_OVERRIDE\*(C' environment variable is true (die otherwise). Last it will run the test mpd with its newly created configuration file.

Everything described above is done automatically when the module is \*(C`use\*(C'-d.

Once the tests are run, the mpd server will be shut down, and the original one will be relaunched (if there was one).

Note that the test mpd will listen to \*(C`localhost\*(C', so you are on the safe side. Note also that the test suite comes with its own ogg files. Those files are 2 seconds tracks recording my voice saying ok, and are freely redistributable under the same license as the code itself.

In case you want more control on the test mpd server, you can use the supplied public methods. This might be useful when trying to test connections with mpd server.

METHODS

customize_test_mpd_configuration( [$port] );

Create a fake mpd configuration file, based on the file mpd.conf.template located in share subdir. The string \s-1PWD\s0 will be replaced by the real path (ie, where the tarball has been untarred), while \s-1TMP\s0 will be replaced by a new temp directory. The string \s-1PORT\s0 will be replaced by $port if specified, 6600 otherwise (\s-1MPD\s0's default). Return the temp dir where the test playlists will be stored.

\fIstart_test_mpd()\fP;

Start the fake mpd, and die if there were any error.

\fIstop_test_mpd()\fP;

Kill the fake mpd.
