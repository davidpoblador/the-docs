SYNOPSIS

    use Regexp::Common qw /list/;

    while (<>) {
        /$RE{list}{-pat => '\w+'}/          and print "List of words";
        /$RE{list}{-pat => $RE{num}{real}}/ and print "List of numbers";
    }

DESCRIPTION

Please consult the manual of Regexp::Common for a general description of the works of this interface.

Do not use this module directly, but load it via Regexp::Common. Returns a pattern matching a list of (at least two) substrings.

If \*(C`-pat=P\*(C' is specified, it defines the pattern for each substring in the list. By default, P is \*(C`qr/.*?\S/\*(C'. In Regexp::Common 0.02 or earlier, the default pattern was \*(C`qr/.*?/\*(C'. But that will match a single space, causing unintended parsing of \*(C`a, b, and c\*(C' as a list of four elements instead of 3 (with \*(C`-word\*(C' being \*(C`(?:and)\*(C'). One consequence is that a list of the form \*(L"a,,b\*(R" will no longer be parsed. Use the pattern \*(C`qr /.*?/\*(C' to be able to parse this, but see the previous remark.

If \*(C`-sep=P\*(C' is specified, it defines the pattern P to be used as a separator between each pair of substrings in the list, except the final two. By default P is \*(C`qr/\s*,\s*/\*(C'.

If \*(C`-lastsep=P\*(C' is specified, it defines the pattern P to be used as a separator between the final two substrings in the list. By default P is the same as the pattern specified by the \*(C`-sep\*(C' flag.

For example:

$RE{list}{-pat=>'\w+'} # match a list of word chars $RE{list}{-pat=>$RE{num}{real}} # match a list of numbers $RE{list}{-sep=>"\t"} # match a tab-separated list $RE{list}{-lastsep=>',\s+and\s+'} # match a proper English list

Under \*(C`-keep\*(C': captures the entire list captures the last separator An alias for $RE{list}{-lastsep=>'\s*,?\s*PATTERN\s*'}

If \*(C`-word\*(C' is not specified, the default pattern is \*(C`qr/and|or/\*(C'.

For example:

$RE{list}{conj}{-word=>'et'} # match Jean, Paul, et Satre $RE{list}{conj}{-word=>'oder'} # match Bonn, Koln oder Hamburg An alias for $RE{list}{conj}{-word=>'and'} An alias for $RE{list}{conj}{-word=>'or'}
