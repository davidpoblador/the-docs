VERSION

This document describes version 2.20 of PostScript::File, released February 11, 2012 as part of PostScript-File version 2.20.

Attributes and methods added since version 2.00 are marked with the version they were added in (e.g. \*(L"(v2.10)\*(R"). Because there were significant \s-1API\s0 changes in 2.00, I recommend that any code using PostScript::File specify a minimum version of at least 2.

SYNOPSIS

Simplest

A 'hello world' program:

    use PostScript::File 2;

    my $ps = PostScript::File->new(reencode => 'cp1252');

    $ps->add_to_page( <<END_PAGE );
        /Helvetica findfont
        12 scalefont
        setfont
        72 300 moveto
        (hello world) show
    END_PAGE

    $ps->output( "test" );

All options

headings => 1, reencode => 'cp1252', font_suffix => '-iso', need_fonts => [qw(Helvetica Helvetica-Bold)],

DESCRIPTION

PostScript::File is a class that writes PostScript files following Adobe's Document Structuring Conventions (\s-1DSC\s0). You should be familiar with the \s-1DSC\s0 if you're using this class directly; consult the PostScript Language Document Structuring Conventions Specification linked to in \*(L"\s-1SEE\s0 \s-1ALSO\s0\*(R".

There are also a number of modules that build upon PostScript::File to produce various kinds of documents without requiring knowledge of PostScript. These are listed in \*(L"\s-1SEE\s0 \s-1ALSO\s0\*(R".

It is possible to construct and output files in either normal PostScript (*.ps files) or as Encapsulated PostScript (*.epsf or *.epsi files). By default a minimal file is output, but support for font encoding, PostScript error reporting and debugging can be built in if required.

Documents can typically be built using only these functions:

new The constructor, with many options add_procset Add PostScript functions to the prolog add_to_page Add PostScript to construct each page newpage Begins a new page in the document output Construct the file and saves it

The rest of the module involves fine-tuning this. Some settings only really make sense when given once, while others can control each page independently. See \*(C`new\*(C' for the functions that duplicate option settings, they all have \*(C`get_\*(C' counterparts. The following provide additional support.

get/set_bounding_box get/set_page_bounding_box get/set_page_clipping get/set_page_landscape set_page_margins get_pagecount draw_bounding_box clip_bounding_box

The functions which insert entries into each of the \s-1DSC\s0 sections all begin with 'add_'. They also have \*(C`get_\*(C' counterparts.

add_comment add_preview add_default add_resource add_procset add_setup add_page_setup add_to_page add_page_trailer add_trailer

Finally, there are a few stand-alone functions. These are not methods and are available for export if requested.

check_tilde check_file incpage_label incpage_roman

Hyphens and Minus Signs

In \s-1ASCII\s0, the character \*(C`\x2D\*(C' (\*(C`\055\*(C') is used as both a hyphen and a minus sign. Unicode calls this character HYPHEN-MINUS (U+002D). PostScript has two characters, which it calls \*(C`/hyphen\*(C' and \*(C`/minus\*(C'. The difference is that \*(C`/minus\*(C' is usually wider than \*(C`/hyphen\*(C' (except in Courier, of course).

In PostScript's StandardEncoding (what you get if you don't use \*(L"reencode\*(R"), character \*(C`\x2D\*(C' is \*(C`/hyphen\*(C', and \*(C`/minus\*(C' is not available. In the Latin1-based encodings created by \*(C`reencode\*(C', character \*(C`\x2D\*(C' is \*(C`/minus\*(C', and character \*(C`\xAD\*(C' is \*(C`/hyphen\*(C'. (\*(C`\xAD\*(C' is supposed to be a \*(L"soft hyphen\*(R" (U+00AD) that appears only if the line is broken at that point, but it doesn't work that way in PostScript.)

Unicode has additional non-ambiguous characters: \s-1HYPHEN\s0 (U+2010), NON-BREAKING \s-1HYPHEN\s0 (U+2011), and \s-1MINUS\s0 \s-1SIGN\s0 (U+2212). The first two always indicate \*(C`/hyphen\*(C', and the last is always \*(C`/minus\*(C'. When you set \*(C`reencode\*(C' to \*(C`cp1252\*(C' or \*(C`iso-8859-1\*(C', those characters will be handled automatically.

To make it easier to handle strings containing HYPHEN-MINUS, PostScript::File provides the \*(L"auto_hyphen\*(R" attribute. When this is true (the default when using \*(C`cp1252\*(C' or \*(C`iso-8859-1\*(C'), the \*(L"pstr\*(R" method will automatically translate HYPHEN-MINUS to either \s-1HYPHEN\s0 or \s-1MINUS\s0 \s-1SIGN\s0. (This happens only when \*(C`pstr\*(C' is called as an object method.)

The rule is that if a HYPHEN-MINUS is surrounded by whitespace, or surrounded by digits, or it's preceded by whitespace or punctuation and followed by a digit, or it's followed by a currency symbol, it's translated to \s-1MINUS\s0 \s-1SIGN\s0. Otherwise, it's translated to \s-1HYPHEN\s0.

ATTRIBUTES

Unlike many classes that use the same method for reading and writing an attribute's value, PostScript::File has separate methods for reading and writing. The read accessor is prefixed with \*(C`get_\*(C', and the write accessor is prefixed with \*(C`set_\*(C'. If no write accessor is mentioned, then the attribute is read-only.

auto_hyphen

$ps->set_auto_hyphen( $translate )

If $translate is a true value, then \*(L"pstr\*(R" will do automatic hyphen-minus translation when called as an object method (but only if the document uses character set translation). (Default: true) See \*(L"Hyphens and Minus Signs\*(R".

If $clipping is true, printing will be clipped to each page's bounding box. This is the document's default value. Each page has its own \*(L"page_clipping\*(R" attribute, which is initialized to this default value when the page is created. (Default: false)

extensions

$ps->set_file_ext( $file_ext )

If $file_ext is undef (the default), then the extension is set automatically based on the output type. \*(C`.ps\*(C' will be added for ordinary PostScript files. \s-1EPS\s0 files have an extension of \*(C`.epsf\*(C' without or \*(C`.epsi\*(C' with a preview image.

If $file_ext is the empty string, then no extension will be added to the filename. Otherwise, it should be a string like '.ps' or '.eps'. (But setting this has no effect on the actual type of the output file, only its name.)

filename

$filename = $ps->get_filename

$ps->set_filename( $file, [$dir] ) An optional fully qualified path-and-file, a simple file name, the empty string (which stands for the special file \*(C`File::Spec->devnull\*(C'), or \*(C`undef\*(C' (which indicates the document has no associated filename). An optional directory name. If present (and $file is not already an absolute path), it is prepended to $file. If no $file was specified, $dir is ignored.

If \*(L"eps\*(R" has been set, multiple pages will have the page label appended to the file name.

Example:

The three pages for user 'chris' on a Unix system would be:

It would be wise to use \*(C`set_page_bounding_box\*(C' explicitly for each page if using multiple pages in \s-1EPS\s0 files.

Paper Size and Margins

These attributes are interrelated, and changing one may change the others.

bounding_box

( $llx, $lly, $urx, $ury ) = $ps->get_bounding_box

$ps->set_bounding_box( $llx, $lly, $urx, $ury )

The bounding box for the whole document. The lower left corner is \*(C`($llx, $lly)\*(C', and the upper right corner is \*(C`($urx, $ury)\*(C'.

Setting the bounding box automatically enables clipping. Call \*(C`$ps->set_clipping(0)\*(C' afterwards to undo that.

The default \*(C`bounding_box\*(C' is calculated from the paper size (taken from the \*(L"paper\*(R", \*(L"height\*(R", and \*(L"width\*(R" attributes) and the \*(L"left\*(R", \*(L"right\*(R", \*(L"top\*(R", and \*(L"bottom\*(R" margins.

Each page also has an individual \*(L"page_bounding_box\*(R", which is initialized from the document's \*(C`bounding_box\*(C' when the page is created.

height

$ps = PostScript::File->new( height => $height )

$height = $ps->get_height

$ps->set_height( $height )

The page height in points. This is normally the longer dimension of the paper. Note that in landscape mode this is actually the width of the bounding box.

Setting this sets \*(L"paper\*(R" to \*(L"Custom\*(R" and the \*(L"bounding_box\*(R" is expanded to the full height.

landscape

$ps = PostScript::File->new( landscape => $landscape )

$landscape = $ps->get_landscape

$ps->set_landscape( $landscape )

If $landscape is true, the page is rotated 90 degrees counter-clockwise, swapping the meaning of height & width. (Default: false)

In landscape mode the coordinates are rotated 90 degrees and the origin moved to the bottom left corner. Thus the coordinate system appears the same to the user, with the origin at the bottom left.

paper

$ps = PostScript::File->new( paper => $paper_size )

$paper_size = $ps->get_paper

$ps->set_paper( $paper_size )

Set the paper size of each page. A document can be created using a standard paper size without having to remember the size of paper using PostScript points. Valid choices are currently A0, A1, A2, A3, A4, A5, A6, A7, A8, A9, B0, B1, B2, B3, B4, B5, B6, B7, B8, B9, B10, Executive, Folio, Half-Letter, Letter, US-Letter, Legal, US-Legal, Tabloid, SuperB, Ledger, 'Comm #10 Envelope', Envelope-Monarch, Envelope-DL, Envelope-C5, and EuroPostcard. (Default: \*(L"A4\*(R")

You can also give a string in the form 'WIDTHxHEIGHT', where \s-1WIDTH\s0 and \s-1HEIGHT\s0 are numbers (in points). This sets the paper size to \*(L"Custom\*(R".

Setting this also sets \*(L"bounding_box\*(R", \*(L"height\*(R", and \*(L"width\*(R" to the full height and width of the paper.

width

$ps = PostScript::File->new( width => $width )

$width = $ps->get_width

$ps->set_width( $width )

The page width in points. This is normally the shorter dimension of the paper. Note that in landscape mode this is actually the height of the bounding box.

Setting this sets \*(L"paper\*(R" to \*(L"Custom\*(R" and the \*(L"bounding_box\*(R" is expanded to the full width.

Page Attributes

The following attributes can have a different value for every page. You can't set them directly in the constructor, but they all have a document-wide default value that each page inherits when it is created. When accessing or setting them, $page is the page label. If $page is omitted, it defaults to the current page.

page_bounding_box

The bounding box for this page. The lower left corner is \*(C`($llx, $lly)\*(C', and the upper right corner is \*(C`($urx, $ury)\*(C'.

Note that calling \*(C`set_page_bounding_box\*(C' automatically enables clipping for that page. If this isn't what you want, call \*(C`$ps->set_page_clipping(0)\*(C' afterwards.

When a page is created, \*(C`page_bounding_box\*(C' is initialized from the document's \*(L"bounding_box\*(R" attribute.

page_clipping

$clipping = $ps->get_page_clipping( [$page] )

$ps->set_page_clipping( [[$page,] $clipping] )

If $clipping is true, printing will be clipped to this page's bounding box. (Default: false)

When a page is created, \*(C`page_clipping\*(C' is initialized from the document's \*(L"clipping\*(R" attribute.

page_label

$ps = PostScript::File->new( page => $page )

$page = $ps->get_page_label

$ps->set_page_label( [$page] )

The label for the current page (used in the %%Page comment). (Default: \*(L"1\*(R")

Unlike the other page attributes, you can only access the \*(C`page_label\*(C' of the current page. (Since pages are specified by label, it makes no sense to ask for the label of a different page.)

When a page is created, \*(C`page_label\*(C' is initialized by passing the previous page's label to the \*(L"incpage_handler\*(R". For the first page, it's initialized from the \*(C`page\*(C' given to the constructor.

page_landscape

$landscape = $ps->get_page_landscape( [$page] )

$ps->set_page_landscape( [[$page,] $landscape] )

If $landscape is true, this page is using landscape mode. (Default: false)

When a page is created, \*(C`page_landscape\*(C' is initialized from the document's \*(L"landscape\*(R" attribute.

METHODS

Note: In the following descriptions, \*(C`[]\*(C' are used to denote optional parameters, not array references.

Constructor

new

$ps = PostScript::File->new(options)

Create a new PostScript::File object, either a set of pages or an Encapsulated PostScript (\s-1EPS\s0) file. Options are hash keys and values. All values should be in the native PostScript units of 1/72 inch.

Example

$ps = PostScript::File->new( eps => 1, landscape => 1, width => 216, height => 288, left => 36, right => 44, clipping => 1, );

This creates an Encapsulated PostScript document, 4 by 3 inch pages printing landscape with left and right margins of around half an inch. The width is always the shortest side, even in landscape mode. 3*72=216 and 4*72=288. Being in landscape mode, these would be swapped. The bounding box used for clipping would then be from (50,0) to (244,216).

\*(C`options\*(C' may be a single hash reference instead of an options list, but the hash must have the same structure. This is more convenient when used as a base class.

File size keys

There are four options which control how much gets put into the resulting file.

debug No debug code is added to the file. Of course there must be no calls to debug functions in the PostScript code. This is the default. db_ functions are replaced by dummy functions which do nothing. A range of functions are added to the file to support debugging PostScript. This switch is similar to the 'C' \*(C`NDEBUG\*(C' macro in that debugging statements may be left in the PostScript code but their effect is removed. Of course, being an interpreted language, it is not quite the same as the calls still takes up space - they just do nothing. See \*(L"\s-1POSTSCRIPT\s0 \s-1DEBUGGING\s0 \s-1SUPPORT\s0\*(R" for details of the functions. Loads the debug functions and gives some reassuring output at the start and a stack dump at the end of each page. A mark is placed on the stack at the beginning of each page and 'cleartomark' is given at the end, avoiding potential \*(C`invalidrestore\*(C' errors. Note, however, that if the page does not end with a clean stack, it will fail when debugging is turned off.

errors

PostScript has a nasty habit of failing silently. If \*(C`errors\*(C' is true, code that prints fatal error messages on the bottom left of the paper is added to the file. For user functions, a PostScript function report_error is defined. This expects a message string on the stack, which it prints before stopping. (Default: true)

headings

If true, add PostScript \s-1DSC\s0 comments recording the date of creation and user's name. (Default: false)

The comments inserted when \*(C`headings\*(C' is true are:

%%For: USER@HOSTNAME %%Creator: Perl module PostScript::File v2.20 %%CreationDate: Sun Jan 1 00:00:00 2012 %%DocumentMedia: US-Letter 612 792 80 ( ) ( )

\s-1USER\s0 comes from \*(C`getlogin() || getpwuid($<)\*(C', and \s-1HOSTNAME\s0 comes from Sys::Hostname. The DocumentMedia values come from the paper size attributes. The DocumentMedia comment is omitted from \s-1EPS\s0 files.

If you want different values, leave \*(C`headings\*(C' false and use \*(L"add_comment\*(R" to add whatever you want.

reencode

Requests that a font re-encode function be added and that the fonts used by this document get re-encoded in the specified encoding. The only allowed values are \*(C`cp1252\*(C', \*(C`iso-8859-1\*(C', and \*(C`ISOLatin1Encoding\*(C'. You should almost always set this to \*(C`cp1252\*(C', even if you are not using Windows.

The list of fonts to re-encode comes from the \*(L"need_fonts\*(R" parameter, the \*(L"need_resource\*(R" method, and all fonts added using \*(L"embed_font\*(R" or \*(L"add_resource\*(R". The Symbol font is never re-encoded, because it uses a non-standard character set.

Setting this to \*(C`cp1252\*(C' or \*(C`iso-8859-1\*(C' also causes the document to be encoded in that character set. Any strings you add to the document that have the \s-1UTF-8\s0 flag set will be re-encoded automatically. Strings that do not have the \s-1UTF-8\s0 flag are expected to be in the correct character set already. This means that you should be able to set this to \*(C`cp1252\*(C', use Unicode characters in your code and the \*(L"-iso\*(R" versions of the fonts, and just have it do the right thing.

Windows code page 1252 (a.k.a. WinLatin1) is a superset of the printable characters in \s-1ISO-8859-1\s0 (a.k.a. Latin1). It adds a number of characters that are not in Latin1, especially common punctuation marks like the curly quotation marks, en & em dashes, Euro sign, and ellipsis. These characters exist in the standard PostScript fonts, but there's no easy way to access them when using the standard or ISOLatin1 encodings. http://en.wikipedia.org/wiki/Windows-1252 <http://en.wikipedia.org/wiki/Windows-1252>

For backwards compatibility with versions of PostScript::File older than 1.05, setting this to \*(C`ISOLatin1Encoding\*(C' re-encodes the fonts, but does not do any character set translation in the document.

Initialization keys

There are a few initialization settings that are only relevant when the file object is constructed.

bottom

The margin in from the paper's bottom edge, specifying the non-printable area. Remember to specify \*(C`clipping\*(C' if that is what is wanted. (Default: 28)

clip_command

The bounding box is used for clipping if this is set to \*(L"clip\*(R" or is drawn with \*(L"stroke\*(R". This also makes the whole page area available for debugging output. (Default: \*(L"clip\*(R").

font_suffix

This string is appended to each font name as it is re-encoded. (Default: \*(L"-iso\*(R")

The string value is appended to these to make the new names.

Example:

$ps = PostScript::File->new( font_suffix => "-iso", reencode => "cp1252" );

\*(L"Courier\*(R" still has the standard mapping while \*(L"Courier-iso\*(R" includes the additional European characters.

left

The margin in from the paper's left edge, specifying the non-printable area. Remember to specify \*(C`clipping\*(C' if that is what is wanted. (Default: 28)

need_fonts

An arrayref of font names required by this document. This is equivalent to calling \*(C`$ps->need_resource(font => @$arrayref)\*(C'. See \*(L"need_resource\*(R" for details.

newpage

(v2.10) Normally, an initial page is created automatically (using the label specified by \*(C`page\*(C'). But starting with PostScript::File 2.10, you can pass \*(C`newpage => 0\*(C' to override this. This makes for more natural loops:

use PostScript::File 2.10; my $ps = PostScript::File->new(newpage => 0); for (@pages) { $ps->newpage; # don't need "unless first page" ... }

It's important to require PostScript::File 2.10 if you do this, because older versions would produce an initial blank page.

If you don't pass a page label to the first call to \*(C`newpage\*(C', it will be taken from the \*(C`page\*(C' option. After the first page, the page label will increment as specified by \*(L"incpage_handler\*(R".

right

The margin in from the paper's right edge. It is a positive offset, so \*(C`right=36\*(C' will leave a half inch no-go margin on the right hand side of the page. Remember to specify \*(C`clipping\*(C' if that is what is wanted. (Default: 28)

top

The margin in from the paper's top edge. It is a positive offset, so \*(C`top=36\*(C' will leave a half inch no-go margin at the top of the page. Remember to specify \*(C`clipping\*(C' if that is what is wanted. (Default: 28)

POSTSCRIPT DEBUGGING SUPPORT

This section documents the PostScript functions which provide debugging output. Please note that any clipping or bounding boxes will also hide the debugging output which by default starts at the top left of the page. Typical \*(C`new\*(C' options required for debugging would include the following.

$ps = PostScript::File->new ( errors => "page", debug => 2, clipcmd => "stroke" );

The debugging output is printed on the page being drawn. In practice this works fine, especially as it is possible to move the output around. Where the text appears is controlled by a number of PostScript variables, most of which may also be given as options to \*(C`new\*(C'.

The main controller is \*(C`db_active\*(C' which needs to be non-zero for any output to be seen. It might be useful to set this to 0 in \*(C`new\*(C', then at some point in your code enable it. Remember that the \*(C`debugdict\*(C' dictionary needs to be selected in order for any of its variables to be changed. This is better done with \*(C`db_on\*(C' but it illustrates the point.

/debugdict begin /db_active 1 def end (this will now show) db_show

At any time, the next output will appear at \*(C`db_xpos\*(C' and \*(C`db_ypos\*(C'. These can of course be set directly. However, after most prints, the equivalent of a 'newline' is executed. It moves down \*(C`db_fontsize\*(C' and left to \*(C`db_xpos\*(C'. If, however, that would take it below \*(C`db_ybase\*(C', \*(C`db_ypos\*(C' is reset to \*(C`db_ytop\*(C' and the x coordinate will have \*(C`db_xgap\*(C' added to it, starting a new column.

The positioning of the debug output is changed by setting \*(C`db_xpos\*(C' and \*(C`db_ytop\*(C' to the top left starting position, with \*(C`db_ybase\*(C' guarding the bottom. Extending to the right is controlled by not printing too much! Judicious use of \*(C`db_active\*(C' can help there.

PostScript functions

x0 y0 x1 y1 cliptobox

This function is only available if 'clipping' is set. By calling the Perl method \*(C`draw_bounding_box\*(C' (and resetting with \*(C`clip_bounding_box\*(C') it is possible to use this to identify areas on the page.

$ps->draw_bounding_box(); $ps->add_to_page( <<END_CODE ); ... my_l my_b my_r my_t cliptobox ... END_CODE $ps->clip_bounding_box();

msg report_error

If 'errors' is enabled, this call allows you to report a fatal error from within your PostScript code. It expects a string on the stack and it does not return.

All the \*(C`db_\*(C' variables (including function names) are defined within their own dictionary (\*(C`debugdict\*(C'). But this can be ignored by all calls originating from within code passed to \*(C`add_to_page\*(C' (usually including \*(C`add_procset\*(C' code) as the dictionary is automatically put on the stack before each page and taken off as each finishes.

any db_show

The workhorse of the system. This takes the item off the top of the stack and outputs a string representation of it. So you can call it on numbers or strings and it will show them. Arrays are printed using \*(C`db_array\*(C' and marks are shown as '--mark--'.

n msg db_nshow

This shows top \*(C`n\*(C' items on the stack. It requires a number and a string on the stack, which it removes. It prints out \*(C`msg\*(C' then the top \*(C`n\*(C' items on the stack, assuming there are that many. It can be used to do a labelled stack dump. Note that if \*(C`new\*(C' was given the option \*(C`debug =\*(C' 2>, There will always be a '--mark--' entry at the base of the stack. See \*(L"debug\*(R".

count (at this point) db_nshow

db_stack

Prints out the contents of the stack. No stack requirements.

The stack contents is printed top first, the last item printed is the lowest one inspected.

array db_print

The closest this module has to a print statement. It takes an array of strings and/or numbers off the top of the stack and prints them with a space in between each item.

[ (myvar1=) myvar1 (str2=) str2 ] db_print

will print something like the following.

myvar= 23.4 str2= abc

When printing something from the stack you need to take into account the array-building items, too. In the next example, at the point '2 index' fetches 111, the stack holds '222 111 [ (top=)' but 'index' requires 5 to get at 222 because the stack now holds '222 111 [ (top=) 111 (next=)'.

222 111 [ (top=) 2 index (next=) 5 index ] db_print

willl output this.

top= 111 next= 222

It is important that the output does not exceed the string buffer size. The default is 256, but it can be changed by giving \*(C`new\*(C' the option \*(C`bufsize\*(C'.

x y msg db_point

It is common to have coordinates as the top two items on the stack. This call inspects them. It pops the message off the stack, leaving x and y in place, then prints all three.

450 666 (starting point=) db_print moveto

would produce:

starting point= ( 450 , 666 )

array db_array

Like \*(L"db_print\*(R" but the array is printed enclosed within square brackets.

var db_where

A 'where' search is made to find the dictionary containing \*(C`var\*(C'. The messages 'found' or 'not found' are output accordingly. Of course, \*(C`var\*(C' should be quoted with '/' to put the name on the stack, otherwise it will either be executed or force an error.

db_newcol

Starts the next debugging column. No stack requirements.

db_on

Enable debug output

db_off

Disable debug output

db_down

Does a 'carriage-return, line-feed'. No stack requirements.

db_indent

Moves output right by \*(C`db_xtab\*(C'. No stack requirements. Useful for indenting output within loops.

db_unindent

Moves output left by \*(C`db_xtab\*(C'. No stack requirements.

EXPORTS

No functions are exported by default. All the functions listed in \*(L"\s-1SUBROUTINES\s0\*(R" may be exported by request.

In addition, the \*(C`pstr\*(C' method may be exported as a subroutine, but this usage is deprecated.

BUGS AND LIMITATIONS

When making \s-1EPS\s0 files, the landscape transformation throws the coordinates off. To work around this, avoid the landscape flag and set width and height differently.

Most of these functions have only had a couple of tests, so please feel free to report all you find.
