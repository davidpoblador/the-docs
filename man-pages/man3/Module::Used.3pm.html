VERSION

This document describes Module::Used version 1.3.0.

SYNOPSIS

    use Module::Used qw< :all >;

    @modules = modules_used_in_files(@files);
    @modules = modules_used_in_modules(@module_names);

    # "strict", "Find::Bin", "warnings"
    @modules = modules_used_in_string(
        'use strict; require Find::Bin; no warnings;'
    );

    # "Exporter"
    @modules = modules_used_in_string( 'use parent 0.221 qw< Exporter >;' );

DESCRIPTION

Modules are found statically based upon \*(C`use\*(C' and \*(C`require\*(C' statements. If use of the base or parent is found, both that module and the referenced ones will be returned. If Moose or Moose::Role are found, this will look for \*(C`extends\*(C' and \*(C`with\*(C' sugar will be looked for; presently, this will miss modules listed in parentheses.

Dynamically loaded modules will not be found.

INTERFACE

Nothing is exported by default, but you can import everything using the \*(C`:all\*(C' tag. Return a list of modules used in the specified files. \*(C`die\*(C's if there is a problem reading a file. Return a list of modules used in the specified modules. \*(C`die\*(C's if there any of the modules weren't found in @INC. Return a list of modules used in the code in the parameter. Return a list of modules used in the specified PPI::Document.

DIAGNOSTICS

Cannot find the location of the named module. Note that this module will not find any dynamically loaded modules. Cannot find the file. Cannot read the file. The \*(L"file\*(R" was actually a directory. \s-1PPI\s0 could not interpret the file as a Perl document.
