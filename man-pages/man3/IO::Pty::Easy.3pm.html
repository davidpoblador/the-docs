SYNOPSIS

    use IO::Pty::Easy;

    my $pty = IO::Pty::Easy->new;
    $pty->spawn("nethack");

    while ($pty->is_active) {
        my $input = # read a key here...
        $input = 'Elbereth' if $input eq "\ce";
        my $chars = $pty->write($input, 0);
        last if defined($chars) && $chars == 0;
        my $output = $pty->read(0);
        last if defined($output) && $output eq '';
        $output =~ s/Elbereth/\e[35mElbereth\e[m/;
        print $output;
    }

    $pty->close;

DESCRIPTION

\*(C`IO::Pty::Easy\*(C' provides an interface to IO::Pty which hides most of the ugly details of handling ptys, wrapping them instead in simple spawn/read/write commands.

\*(C`IO::Pty::Easy\*(C' uses IO::Pty internally, so it inherits all of the portability restrictions from that module.

METHODS

new(%params)

The \*(C`new\*(C' constructor initializes the pty and returns a new \*(C`IO::Pty::Easy\*(C' object. The constructor recognizes these parameters:

handle_pty_size

A boolean option which determines whether or not changes in the size of the user's terminal should be propageted to the pty object. Defaults to true.

def_max_read_chars

The maximum number of characters returned by a \*(C`read()\*(C' call. This can be overridden in the \*(C`read()\*(C' argument list. Defaults to 8192.

raw

A boolean option which determines whether or not to call \*(L"set_raw()\*(R" in IO::Pty after \*(C`spawn()\*(C'. Defaults to true.

spawn(@argv)

Fork a new subprocess, with stdin/stdout/stderr tied to the pty.

The argument list is passed directly to \*(C`exec()\*(C'.

Returns true on success, false on failure. Read data from the process running on the pty.

\*(C`read()\*(C' takes two optional arguments: the first is the number of seconds (possibly fractional) to block for data (defaults to blocking forever, 0 means completely non-blocking), and the second is the maximum number of bytes to read (defaults to the value of \*(C`def_max_read_chars\*(C', usually 8192). The requirement for a maximum returned string length is a limitation imposed by the use of \*(C`sysread()\*(C', which we use internally.

Returns \*(C`undef\*(C' on timeout, the empty string on \s-1EOF\s0, or a string of at least one character on success (this is consistent with \*(C`sysread()\*(C' and Term::ReadKey). Writes a string to the pty.

The first argument is the string to write, which is followed by one optional argument, the number of seconds (possibly fractional) to block for, taking the same values as \*(C`read()\*(C'.

Returns undef on timeout, 0 on failure to write, or the number of bytes actually written on success (this may be less than the number of bytes requested; this should be checked for).

is_active

Returns whether or not a subprocess is currently running on the pty. Sends a signal to the process currently running on the pty (if any). Optionally blocks until the process dies.

\*(C`kill()\*(C' takes two optional arguments. The first is the signal to send, in any format that the perl \*(C`kill()\*(C' command recognizes (defaulting to \*(L"\s-1TERM\s0\*(R"). The second is a boolean argument, where false means to block until the process dies, and true means to just send the signal and return.

Returns 1 if a process was actually signaled, and 0 otherwise.

close

Kills any subprocesses and closes the pty. No other operations are valid after this call.

handle_pty_size

Read/write accessor for the \*(C`handle_pty_size\*(C' option documented in the constructor options.

def_max_read_chars

Read/write accessor for the \*(C`def_max_read_chars\*(C' option documented in the constructor options.

pid

Returns the pid of the process currently running in the pty, or undef if no process is running.
