SYNOPSIS

xdx [OPTION]...

DESCRIPTION

Xdx is a GUI DX-cluster client which shows a list with DX announcements and a separate text widget with WWV, WCY, "To ALL" and other server messages in a single Gtk+ window. DX Cluster hosts are remembered between sessions.

OPTIONS

Here is a summary of the supported options:

-c, --cty_dat=path/to/cty.dat

Set path for desired country file.

-h, --help

Print short xdx usage message detailing available options and exit.

-V, --version

Print Xdx version and short copyright information and exit.

ENVIRONMENT

Instead of the -c or --cty_dat command line options, Xdx will honor the XDX_CTY environment variable. However, when the command line option is given the environment varriable will be ignored.

FILES

Xdx will look for the country file in the preferences directory, normally $HOME/.xdx, if a path is not given on the command line or the XDX_CTY environment variable is not set. This is a convenient way to keep an up-to-date or custom cty.dat for use just with Xdx.

NOTES

Updated country files are available from: http://www.country-files.com/cty/cty.dat

Development is currently hosted at: https://github.com/N0NB/xdx

Xdx has been translated into several languages. More translations are welcome. Please contact the maintainer for details.

BUGS

Xdx has a new maintainer, Nate Bargmann, N0NB. Please report bugs to <xdxclusterclient-discuss@lists.sourceforge.net>.
