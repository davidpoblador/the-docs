SYNOPSIS

viewmount [-hr] [-t vfstype ] [-o options ] device dir

DESCRIPTION

This command has the same meaning and and a reduced set of features of mount(8). View-OS previously used the standard mount command from linux-utils-ng. Since v.2.15.rc1 mount is not compatible any more with View-OS for an unnecessary restriction when the executable is not setuid. We hope the developers of linux-utils-ng can fix the problem asap.
