SYNOPSIS

\*(C`simple2zoom\*(C' [ \*(C`-c\*(C' configFile ] [ \*(C`--\*(C' YAZ-options ] [ listener-address ... ]

DESCRIPTION

\*(C`simple2zoom\*(C' provides a generic gateway between the Z39.50 and \s-1SRU/SRW\s0 Information Retrieval protocols. Because it relies on the \*(C`Net::Z3950::SimpleServer\*(C' and \*(C`ZOOM\*(C' modules for the server and client funcationality respectively, because both of these modules are based on the \s-1YAZ\s0 toolkit, and because \s-1YAZ\s0 transparently handles all three standard \s-1IR\s0 protocols (\s-1ANDI/NISO\s0 Z39.50, \s-1SRU\s0 and \s-1SRW\s0), it can function as a gateway from any and to any of these protocols. However, its principle purpose is as a Z39.50 server that proxies requests through to a backend by acting as an \s-1SRU\s0 client.

The following command-line options govern how the gateway functions:

-c configFile

Specifies that the named configFile should be used to configure the functionality of the gateway: if this option is not specified, then the file \*(C`client.xml\*(C' is used. The format of the configuration file is described separately in \*(C`Net::Z3950::Simple2ZOOM::Config\*(C', and a sample configuration file, \*(C`test.xml\*(C', is supplied in the \*(C`etc\*(C' directory of the distribution.

--

Indicates the end of \*(C`simple2zoom\*(C'-specific options. This is required if \s-1YAZ\s0 options are to be specified, so that \*(C`simple2zoom\*(C' doesn't try to interpret them itself.

YAZ-options

Command-line arguments subsequent to the \*(C`--\*(C' option are interpreted by the \s-1YAZ\s0 backend server as described at http://www.indexdata.dk/yaz/doc/server.invocation.tkl These options provide the means to control many aspects of the gateway's functioning: for example, whether the server forks a new process for each client or runs a single process using \*(C`select()\*(C'; how (if at all) to interpret incoming \s-1SRU\s0 requests; whether and how to log protocol packets for debugging.

listener-address

One or more YAZ-style listener addresses may be specified, and the server will accept connections on those addresses: for example, @:9998, \*(C`unix:/tmp/somesocket\*(C' or \*(C`ssl:myhost.com:210\*(C'. If no explicit listener addresses are provided, the server listens on port 9999.
