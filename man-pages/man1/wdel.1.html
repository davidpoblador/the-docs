SYNOPSIS

wdel [option]... [\s-1URL\s0 [\s-1file\s0]...]...

DESCRIPTION

Wdel is a free utility that is able to delete files on an ftp server.

Wdel is non-interactive and background-capable. It can delete files or whole directories and is meant to be a robust client even for unstable connections and will therefore retry to delete a file, if the connection broke.

Wdel supports connections through proxies, allowing you to use it in an environment that can access the internet only via a proxy or to provide anonymity by hiding your ip-address to the server. For SOCKSv5-proxies Wdel supports also listening mode, allowing you to use port-mode ftp through a proxy (useful if the remote ftp is behind a firewall or a gateway).

URL-Input-Handling

URLs are recognized by the ftp://-prefix.

Wdel first reads the URLs from the command line, and associates every file with the last URL until a new URL is found (note that this is different from wput). Then, it reads the --input-file (if any) and parses the URLs in the same way. So you can specify e.g. one URL and read all filenames from a file.

If there are no explicit filenames given for a singe URL, then the last part of the URL is taken to be the file/directory to be deleted. If there are filenames, then the last part of the URL is considered to be the directory where these files reside in. Slashes at the end of the URL or the filename (in case it is a directory to be deleted) are ignored.
