SYNOPSIS

orbit-name-server-2 [--key=string] [-?] [--help] [--usage]

DESCRIPTION

orbit-name-server-2 is an ORBit2 based implementation of the OMG's COS Naming Service Specification.

When orbit-name-server-2 starts up successfully it writes the stringified object reference for its root context on standard output.

OPTIONS

orbit-name-server-2 accepts the following options:

--key=string

Respond to corbaloc requests with this object key.

-? --help

Show a help message.

--usage

Display a brief usage message.

FILES

On startup, orbit-name-server-2 tries to read the files /etc/orbitrc and $HOME/.orbitrc. The format of these files is not yet documented in Debian.
